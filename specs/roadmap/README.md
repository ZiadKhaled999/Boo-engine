# Boo Engine Roadmap

## Principles
- Android-first, local-first, plugin-based.
- Build vertical slices before expanding APIs.
- One source of truth: specs drive implementation.

## Phase Strategy
1. **Phase 1**: Core runtime proof (APK + bridge + offline demo).
2. **Phase 2**: First native capabilities (filesystem/notifications).
3. **Phase 3**: Developer experience (CLI, docs, templates).
4. **Phase 4**: Plugin ecosystem (contracts and tooling).
5. **Phase 5**: Advanced runtime features.
6. **Phase 6**: Hardening and governance.

## Exit Criteria (applies to every phase)
- Working demo and reproducible setup.
- Tests for happy-path and denial-path.
- Security review notes.
- Updated docs/specs.
